### Reporting Bugs ğŸ›

If you find a bug, please create an issue with:
- A clear description of the problem
- Steps to reproduce the issue
- Expected vs actual behavior
- Your environment (OS, Bash version, Git version)

### Suggesting Features ğŸ’¡

We welcome feature suggestions! Please create an issue describing:
- The feature you'd like to see
- Why it would be useful
- How you envision it working

### Submitting Code ğŸ”§

1. Fork the repository
2. Create a new branch for your feature (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test your changes thoroughly
5. Commit with clear, descriptive messages
6. Push to your fork
7. Open a Pull Request

### Code Style

- Use clear, descriptive variable names
- Add comments for complex logic
- Follow existing code formatting
- Keep functions focused and modular
- Handle errors gracefully

### Testing

Before submitting a PR:
- Test all menu options
- Verify error handling works correctly
- Test in different scenarios (new repo, existing repo, etc.)
- Ensure the script runs on different systems if possible
